## What Was Built

### 1. `log_selector` Profile Type (capacity.js)

Added a new capacity profile type `log_selector` for solutions that carry a `log_types[]` array in solutions.json. Detection happens in `getConnectorCapacityMetadata()` — any solution with a non-empty `log_types` array returns `type: 'log_selector'` instead of `type: 'none'`. This causes AWS to appear in the sizing drawer with a dedicated UI.

Supporting functions added:
- `createDefaultSizingDraft('log_selector')` — returns `{ selectedLogTypes: [...default_selected ids], isDefault: true }`
- `normalizeSizingValues('log_selector')` — normalizes `selectedLogTypes` array
- `validateSizingDraft('log_selector')` — always valid (any selection including empty is acceptable)
- `computeSizingResult('log_selector')` — returns summary string `"N log types selected"`
- `updateConnectorCapacityEntries()` — extended to handle `log_selector` (stores `{ selectedLogTypes, isDefault }` per solution)
- `getSolutionCapacityProfile()` — adds `logTypes` field to profile when `type === 'log_selector'`

Sizing is persisted per-solution (same mechanism as firewall/linux EPS), keyed by solution ID.

### 2. Log-Type Selector UI (solutions.js)

Added an early-return branch at the top of `createSizingEditor()` for `profile.type === 'log_selector'`. The UI renders:
- Intro text explaining the auto-save behavior
- Checkbox list (`log-type-selector` CSS class) — one row per log type
- Each row: checkbox + label + analytics badge ("61 rules" / "raw only") + infrastructure label
- Warning message for raw-only selections (hidden when not applicable)
- Note explaining the Gantt impact

Auto-save behavior: on every checkbox `change` event, saves to localStorage via `saveConnectorCapacityValues` and updates the solution card badge via `syncSolutionCardState` — **without re-rendering the drawer**, preserving focus and avoiding jank.

`createSizingDrawerOverview()` updated to show subtitle "Log source selection" for `log_selector` type.

### 3. Dynamic Gantt Task Generation (gantt-tasks.js)

`buildPerConnectorTasks()` now accepts `capacitySnapshot` in its options object. When a connector has `log_types[]`, it:
1. Reads `selectedLogTypes` from the snapshot (falls back to `default_selected` ids)
2. Filters to `selectedTypes`
3. If `selectedTypes.length > 0` → calls `buildLogTypeTasks()` for AWS-specific task set
4. If empty → falls through to the generic minimal-pack 1-task path (fallback)

`buildLogTypeTasks()` generates:
- `{abbrev}-IAM` — IAM prerequisites (0.5d, always)
- Per selected type: `{abbrev}-{CODE}-CFG` (S3+SQS setup, `setup_duration_days` from data) + `{abbrev}-{CODE}-VAL` (validate ingestion, 0.25d)
- `{abbrev}-CONTENT` — deploy analytics rules (only if any selected type has `has_content: true`; duration = `ceil(max(4h, rules × 5min) / 4) × 4`)
- `{abbrev}-TUNE` — tune detections (0.5d, only when content deployed)

Content/tuning are **skipped entirely** when all selected types are raw-only. CFG tasks depend on IAM (can run in parallel); VAL depends on its own CFG; CONTENT depends on all VALs.

Duration math verification:
- CloudTrail + GuardDuty (defaults): 4h + 4h + 4h + 2h + 2h + 8h + 4h = 28h = **3.5d** ✓
- CloudTrail only: 4h + 4h + 2h + 8h + 4h = 22h = **2.75d** ✓
- VPC Flow only (raw): 4h + 4h + 2h = 10h = **1.25d** ✓

AWS gets abbreviation `AWS` in `KNOWN_ABBREVS`.

Task IDs use a `LOG_TYPE_TASK_CODES` lookup (`cloudtrail → CTL`, `guardduty → GDT`, `vpcflow → VPF`, `cloudwatch → CWL`, `custom → CSL`) with a generic 2+2 char fallback for future log_types additions.

`buildGanttPlan()` updated to pass `capacitySnapshot` to `buildPerConnectorTasks()`.

### 4. CSS (style.css)

Added `.log-type-selector`, `.log-type-selector__item`, `.log-type-selector__name`, `.log-type-selector__badge`, `.log-type-selector__badge.is-raw`, `.log-type-selector__infra`, `.log-type-selector__warning` — dark-theme styles consistent with existing sizing drawer patterns.

## Key Decisions

| Decision | Rationale |
|---|---|
| `log_selector` as new profile type, not a variant of `firewall` | Clean separation; `firewall` has EPS/VM math, `log_selector` has no VM sizing. Avoids polluting existing validation paths. |
| Auto-save on checkbox change without drawer re-render | User-facing immediacy (like task spec requested). Targeted DOM update (`syncSolutionCardState`) is faster and preserves focus. |
| Defaults from `default_selected` when nothing is saved | Ensures sensible Gantt output even before user opens drawer. CloudTrail + GuardDuty are always generated by default. |
| Fall through to generic minimal-pack path when zero types selected | Preserves the existing fallback for edge cases while dynamic generation takes over the normal path. |
| `log_types` feature is solution-generic, not AWS-specific | All detection code checks `Array.isArray(solution.log_types)`. Any future solution with `log_types[]` gets the same UI and Gantt behavior automatically. |
| Content/tuning tasks gated on `has_content: true` | Raw-only selections (VPC Flow, CloudWatch, Custom) should not generate content deployment tasks — no built-in rules to deploy. |
